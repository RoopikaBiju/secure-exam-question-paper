How Blockchain Secures Exams
3-layer security model
Why This System Cannot Be Tampered With
Three independent security layers work together to make exam paper tampering cryptographically impossible.
🔐 Layer 1 — AES-256-GCM Encryption
The exam paper is encrypted using AES-256-GCM before storage on the server. GCM mode produces an authentication tag alongside the ciphertext. If even a single byte of the encrypted file is modified by an attacker, the tag verification fails instantly — making decryption mathematically impossible, even with the correct key.
⛓️ Layer 2 — Blockchain Integrity
The SHA-256 hash of the original paper is permanently written to the Ethereum blockchain at the time of upload. This record cannot be altered or deleted. Anyone — student, faculty, or examiner — can verify the paper's authenticity at any time by comparing the computed hash with the blockchain record, with no login required.
⏰ Layer 3 — Time-Locked Key Release
The AES decryption key is stored inside the smart contract and is programmatically locked until the scheduled exam time. The release is enforced by the blockchain itself — not by a human. Even the admin cannot bypass this. Pre-exam leakage of the decryption key is impossible.
What Happens If Someone Tries to Tamper
Here is exactly what happens if an attacker modifies the encrypted file on the server:
STEP 1 Attacker modifies bytes in the encrypted file on server
STEP 2 Exam center downloads the modified file
STEP 3 System attempts AES-GCM decryption with correct key
RESULT ❌ Authentication tag mismatch — decryption FAILS completely
RESULT ❌ SHA-256 hash mismatch — blockchain proof confirms tampering
RESULT ❌ Paper is unreadable — attacker achieved nothing
✅ The original encrypted paper on the blockchain remains the ground truth
